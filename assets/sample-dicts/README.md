# Sample dictionaries

Three concrete examples — one per supported format — that you can copy as-is to verify your device picks up sideloaded dictionaries, and as a starting point for authoring your own.

| File / folder | Format | Layout | Display name in popup |
|---|---|---|---|
| [`cooking-terms.csv`](cooking-terms.csv) | CSV | flat (single file at root) | `cooking-terms` |
| [`supernote-terms.json`](supernote-terms.json) | JSON (object-map) | flat (single file at root) | `supernote-terms` |
| [`sn-tech-jargon/`](sn-tech-jargon/) | StarDict | organised (subfolder, with `meta.json`) | `Tech Jargon` |

Drop any of these into `MyStyle/SnDict/` on your Supernote and the plugin will pick them up at next launch. Both layouts can coexist — flat single-file dicts at the root and organised subfolder dicts, side by side.

The StarDict sample is regenerated by `npm run build:sample-dicts` from entries in `scripts/buildSampleDicts.mjs`. The CSV and JSON samples are committed as-is — open them in any text editor and adapt.
